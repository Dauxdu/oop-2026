## Что такое vcpkg

vcpkg — это кроссплатформенный менеджер зависимостей для C и C++, разработанный Microsoft. Его основная задача — избавить разработчиков от ручной настройки сторонних библиотек на разных операционных системах.

Представим ситуацию: ты начинаешь новый проект или подключаешься к существующему. Без менеджера пакетов пришлось бы вручную искать исходники библиотек, настраивать пути компиляции, решать конфликты версий и повторять этот процесс на каждой машине. vcpkg автоматизирует поиск, загрузку, сборку и интеграцию библиотек, делая процесс воспроизводимым и предсказуемым.

### Режим манифеста

Режим манифеста — это рекомендуемый способ работы с vcpkg. В этом режиме зависимости проекта описываются в файле `vcpkg.json`, аналогично `requirements.txt` в Python или `package.json` в Node.js.

Благодаря этому любой разработчик может получить идентичное окружение, выполнив одну команду. Зависимости устанавливаются локально (можно зафиксировать конкретные версии библиотек) в каталог проекта, что исключает конфликты версий между разными проектами.

Все эти настройки хранятся в папке с проектом

## Установка vcpkg

Процесс установки зависит от операционной системы, но общая логика едина: клонирование репозитория, выполнение скрипта инициализации и настройка переменных окружения.

### Предварительные требования

Перед установкой убедитесь, что в системе установлены:

- Git
- Компилятор C++ (MSVC на Windows, GCC/Clang на Linux и macOS)
- CMake

### Windows

1. Выберите папку для установки. Рекомендуемый путь: `C:\vcpkg`

2. Создайте папку, откройте терминал от имени администратора и выполните:

```powershell
git clone https://github.com/microsoft/vcpkg C:\vcpkg
cd C:\vcpkg

.\bootstrap-vcpkg.bat -disableMetrics
```

> [!NOTE]
> Параметр `-disableMetrics` отключает сбор анонимной телеметрии.

3. Создайте переменную `VCPKG_ROOT` и добавьте vcpkg в переменную `PATH`:

```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\vcpkg", "Machine")
[Environment]::SetEnvironmentVariable("VCPKG_ROOT", "C:\vcpkg", "Machine")
```

Изменения вступают в силу по мере новых процессов системы. Рекомендуется перезапустить операционную систему.

### Linux

1. Установите зависимости:

Для Debian/Ubuntu

```bash
sudo apt update && sudo apt install git curl zip unzip tar vcpkg
```

Для Arch-like Linux

```bash
sudo pacman -Syu git curl zip unzip tar vcpkg
```

2. Склонируйте и инициализируйте vcpkg:

```bash
git clone https://github.com/microsoft/vcpkg $VCPKG_ROOT
cd $VCPKG_ROOT

./bootstrap-vcpkg.sh -disableMetrics
```

### macOS

1. Установите [homebrew](https://brew.sh/) — менеджер пакетов для MacOS.

2. Установите vcpkg:

```bash
brew install vcpkg
```

2. Склонируйте и инициализируйте vcpkg:

```bash
git clone https://github.com/microsoft/vcpkg $VCPKG_ROOT
cd $VCPKG_ROOT

./bootstrap-vcpkg.sh -disableMetrics
```

### Инициализация манифеста

В корне вашего проекта выполните:

```bash
vcpkg new --application
```

Команда создаст два файла:

- `vcpkg.json` — описывает прямые зависимости проекта.
- `vcpkg-configuration.json` — управляет источниками пакетов и базовыми версиями (baseline), что позволяет «заморозить» состояние репозитория и избежать неожиданных обновлений

Пример `vcpkg.json`:

```json
{
  "dependencies": ["sfml", "boost"]
}
```

### Добавление зависимостей

Используйте команду `add port` для добавления библиотек:

```bash
vcpkg add port sfml boost
```

### Интеграция с системами сборки

**CMake**

Передайте путь к toolchain-файлу vcpkg при конфигурации:

```bash
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake
```

### Использование системных пакетов (Linux, macOS)

Технически проект можно собрать, установив зависимости через системный менеджер (`pacman`, `apt`, `brew`). Однако такой подход имеет недостатки:

- сложно фиксировать версии библиотек
- различия в путях и конфигурациях между дистрибутивами
- отсутствие единого интерфейса для всех платформ

vcpkg устраняет эти проблемы, предоставляя согласованное поведение на всех ОС.

### Экспорт зависимостей

Для передачи скомпилированных библиотек в другие окружения используйте команду экспорта:

```bash
vcpkg export sfml boost --zip --output-dir=./vcpkg_sdks
```

Команда создаст архив с готовыми к использованию библиотеками, заголовочными файлами и CMake-конфигурациями.

### Отладка и диагностика

- Просмотр установленных пакетов: `vcpkg list`
- Поиск доступных портов: `vcpkg search <имя>`
- Информация о пакете: `vcpkg info <порт>`
